# Workflow

## Scope
Use this workflow for greenfield Expo Router + TypeScript iOS-first delivery where final output must include implemented product features, not scaffold-only placeholders.

## Contract Hierarchy
Use this precedence when conflicts appear:
1. `SKILL.md`
2. completed project PRD based on `Prd Template.md`
3. `references/prd-mapping.md`
4. `references/feature-modules.md`
5. `references/testing-quality.md`
6. `references/eas-testflight.md`
7. user request details that do not violate higher contracts

## Required Inputs
- `AppName`
- `BundleId`
- `OutputDir`
- `PrdPath`

## Lifecycle Phases
### Phase 0: Preflight
- Verify greenfield scope and writable output path.
- Validate PRD required sections/placeholders.
- Derive scaffold flags from PRD Section 3.
- Verify Node/npm/npx availability and compatibility.

### Phase 1: Scaffold Baseline
```powershell
./scripts/scaffold_expo_ios_app.ps1 -AppName weather-app -BundleId com.example.weatherapp -OutputDir C:/work -WithUiFoundation -WithAuth
```

### Phase 2: Bootstrap Requirement Mapping
```powershell
py ./scripts/bootstrap_prd_implementation.py --project-dir C:/work/weather-app --prd-path C:/work/PRD.md
```

### Phase 3: Implement PRD Features
- Implement P0 features and journeys in app code.
- Replace placeholder/starter content generated by templates.
- Update `reports/prd-implementation.json` with `status`, `code[]`, and `tests[]`.

### Phase 4: Contract Validation
```powershell
py ./scripts/validate_expo_ios_project.py --project-dir C:/work/weather-app --prd-path C:/work/PRD.md --report-path C:/work/weather-app/reports/validator.pre-ci.json
```

### Phase 5: CI and EAS Setup
```powershell
./scripts/setup_ci_eas.ps1 -ProjectDir C:/work/weather-app -RepoProvider github -ReleaseBranch main
py ./scripts/validate_expo_ios_project.py --project-dir C:/work/weather-app --prd-path C:/work/PRD.md --report-path C:/work/weather-app/reports/validator.post-ci.json
```

### Phase 6: Quality Gates
```powershell
npm run lint
npm run typecheck
npm run test
```

### Phase 7: Release Readiness
- Complete `release/human-inputs.md`.
- Continue with `references/eas-testflight.md`.
- Run EAS build/submit only after all blocker checks pass.

## Stop Conditions
- Required inputs missing.
- PRD contract failure (`BLOCKED_INPUT`).
- Placeholder marker check fails.
- P0 requirement mapping/evidence check fails.
- Blocker gate fails after allowed retries.

## Exit Criteria
- Validator exits `0` with `--prd-path`.
- `reports/prd-implementation.json` covers all PRD requirements.
- P0 requirements are implemented and verified by tests.
- Placeholder scan has zero findings.
- CI workflow exists and quality gates pass.
